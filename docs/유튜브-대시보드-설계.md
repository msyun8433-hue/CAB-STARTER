## 4. 작동 방식
```
매일 08:00 (KST)
  → Vercel Cron(자동 타이머)이 수집 함수를 깨움
  → YouTube Data API v3 로 채널 현재 수치 + 최근 영상 조회
  → 스냅샷(그날의 숫자)을 저장소에 기록
  → 어제 스냅샷과 비교해 증감 계산
대시보드 접속 시
  → 저장된 최신 스냅샷 + 추세를 카드 UI로 렌더링
```

## 5. 기술 구성
- **프런트+백**: Next.js (App Router) — 한 프로젝트에 화면과 수집 함수 공존
- **자동 타이머**: Vercel Cron — `0 23 * * *` UTC = 08:00 KST
- **데이터 소스**: YouTube Data API v3 (`channels.list`, `search.list`, `videos.list`)
- **저장소(스냅샷 누적)**: Supabase(무료) 테이블 `daily_snapshots`
  - 컬럼: id, channel_id, date, subscribers, total_views, total_videos, raw(json)
  - 영상 단위: `video_snapshots` (video_id, date, views, likes)
- **배포**: Vercel (사이트 폴더만 게시 — 사업 폴더·.env·고객정보 제외)

## 6. 필요한 비밀값 (.env 에만)
- `YOUTUBE_API_KEY` — 구글 클라우드에서 발급 (사용자 손작업)
- `YOUTUBE_CHANNEL_ID` — 채널 주소에서 추출
- `SUPABASE_URL`, `SUPABASE_SERVICE_KEY` — 저장소 연결
- `CRON_SECRET` — 수집 함수 무단 호출 방지

## 7. 비용
- YouTube API: 무료 할당량(하루 10,000 단위) 안에서 충분
- Vercel / Supabase: 이 규모면 무료 플랜 범위
