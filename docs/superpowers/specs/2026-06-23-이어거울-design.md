# 이어 거울 — 설계 (2단 AI거울 · 첫 루프)

> 작성: 2026-06-23 · brainstorming 스킬로 도출, 윤미선 승인.
> 한 줄: 고객이 세션에 적어 보낸 기록을, 이어가 **세 결의 짧은 편지**로 비춰 돌려준다. 답·조언·평가는 없다.
> 근거: `자료함/★지금-기준.md` "🦴 사업의 척추 — 거울이지 코치가 아니다" / `docs/고객여정_설계.md` ⑤AI 거울(2단).

---

## 1. 목적과 범위

- **목적:** 한 번도 안 돌려본 2단 AI거울을 **첫 루프** 돌려, "거울 글이 좋은가"를 검증한다.
- **이번 범위(첫 루프):** 수동 붙여넣기 방식. 메일 자동 읽기·자동 발송·supabase 저장·한 권 쌓기는 **이번에 안 함**(글이 검증된 뒤).
- **만들 물건:** 슬래시 명령 `/mirror` 하나. 프로젝트 `.claude/commands/`에 둠(USB 따라감).

## 2. 흐름

```
고객이 session.html에 적음 → 보내기 → 윤미선 지메일 도착
   → 윤미선이 /mirror 치고 기록 텍스트 붙여넣기
   → 이어가 거울 편지 생성 → AI 말투 빼기 → [결다듬기] 통과
   → 윤미선에게 보여줌 → 윤미선이 읽고(필요시 손봄) → 직접 답장 발송
```

§0(몸 안 갈림)·§10(메일 발송은 윤미선이 누름)을 둘 다 지킨다. AI는 글까지만, 발송은 사람이.

## 3. 거울 편지의 세 결

한 편의 짧은 편지처럼 자연스럽게 흐른다(항목 나열식 X).

| 결 | 하는 일 | 안 하는 일 |
|---|---|---|
| ① 정돈 | 고객이 쓴 말을 **그의 언어 그대로** 모아 되비춤(요약·구조화) | 새 단어로 바꿔 말하기 |
| ② 비춤 | 반복되는 결/실 **한 가지**를 짚음 + 깃털처럼 **한 번만** 의미 쪽으로 살짝 기울임 | 해석으로 빠지기, "이건 ~뜻이에요" |
| ③ 다음 한 걸음 | 고객이 **이미 적은** '내일의 작은 행동(7번)'을 되돌려줌 | 새 과제·조언 주기 |

### ②비춤의 "살짝선" (가장 조심할 경계)
- **OK:** "버팀·잘 버텼다·또 참았네 — 버티고 참는 결이 몇 번 지나가요" + 괄호로 깃털 한 번("무겁다고만 하기엔, 꽤 오래 받치고 있었던 것 같기도 하고요").
- **금지:** "이건 당신이 인정받지 못해서예요"(원인 단정) / "이렇게 해보세요"(처방) / "당신은 외로운 거예요"(감정 이름 단정).
- 순수 되돌림만 하면 의미가 약하니 **딱 한 번** 의미 쪽으로 기울이고 멈춘다(윤미선 지침 2026-06-23).

### 거울 편지 예시 (샘플 입력 → 출력)
> 입력: 감각=무거움·어깨 뻐근 / 한 단어=버팀 / 장면=회의에서 의견 묻힘, 가슴 조임, "또 참았네" / 목소리A=괜찮은 척 / 목소리B=사실 인정받고 싶다 / 한 문장=오늘의 나는 잘 버텼다 / 내일=자기 전 한 문장 적기

```
○○님,

오늘 당신은 이렇게 적었어요. 몸은 무겁고 어깨가 뻐근했고,
한 단어로는 '버팀'. 회의에서 의견이 묻힌 장면이 오래 남았고,
그때 가슴이 조였다고. 한쪽에선 괜찮은 척하면서, 다른 쪽에선
인정받고 싶다고 적었고요. 그래도 오늘의 나는 잘 버텼다, 고 끝맺었어요.

'버팀'이라는 한 단어와 '잘 버텼다'는 문장, '또 참았네'라는 장면 —
오늘 당신의 말에는 버티고 참는 결이 몇 번 지나가요.
(무겁다고만 하기엔, 꽤 오래 어깨로 받치고 있었던 것 같기도 하고요.)

당신은 내일 '자기 전 한 문장 적기'를 건네기로 했어요.
그 한 문장이 오늘처럼 버틴 이야기일지 아닐지 — 내일의 당신이 적어줄 거예요.

— 이어
```

## 4. 안전장치 (핵심)

1. **AI 말투 빼기(명시 단계):** 같은 문장 구조 반복·과한 접속사("그리고/또한/게다가" 남발)·"~하시면 됩니다" 설명체·이모지·번호 나열 금지. 윤미선의 담백·정갈한 결.
   - 도구: `blader/humanizer`(voice matching) — 윤미선 문체 샘플에 맞춰 다시 씀. (설치 확인 받는 중. 없어도 첫 루프는 [결다듬기]+이 규칙으로 작동.)
2. **[결다듬기] 스킬 통과:** 나가기 전 금지어(치유·회복·성장·명상·소마틱·코칭·클래스…)·과장·압박 문구 0 확인 (CLAUDE.md §5).
3. **금지 동작 못박기:** 평가·조언·해결책·미래 처방·"이렇게 살아라"·원인 단정·감정 이름 단정 금지.
4. **빈 칸 처리:** 고객이 안 적은 항목은 지어내지 않고 건너뛴다 (추측·창작 금지, §9).
5. **호칭:** 고객이 '불리고 싶은 이름'으로. 없으면 호칭 없이.

## 5. `/mirror` 명령 동작

- **이름:** `mirror` (영어 — 한글 /명령은 "/"목록에서 안 먹힘, 이어가기.md 확인).
- **입력:** 명령 친 뒤, 지메일에서 복사한 기록 텍스트를 붙여넣음.
- **처리:** 위 3·4 규칙대로 세 결의 편지 생성 → AI 말투 빼기 → [결다듬기] 통과 → 윤미선에게 보여줌.
- **출력:** 거울 편지 1편(붙여 보내기 좋은 형태). 발송은 윤미선이 직접.

## 6. 첫 루프 "됐다" 기준

베타 기록 1개로 한 바퀴 돌려서:
1. 금지어·과장 0
2. 고객의 언어로 정돈됨(새 단어로 바꾸지 않음)
3. 패턴 한 가지가 과하지 않게 짚임(②비춤의 살짝선 안)
4. 답·조언·평가 없음
5. 윤미선이 "이건 나가도 되겠다" 느낌

## 7. 안 하는 것 (이번 범위 밖 — 10명 뒤)

- 한 권 쌓기·그달 제목 자동 제안 — ⑥단계, 10명 뒤
- 가지치기(개인화 분기) 설계 — 실제 기록 보고 (★지금-기준.md)
