# 〈하루 끝〉 세션 현황판 — 설계 문서 (v2)

> 작성: 2026-06-21 · 갱신: 2026-06-21 (v2 — 오늘 칸·진척·병목·추천 추가) · 상태: 검토 대기
> 관련 원칙: CLAUDE.md §0(도구가 일하는 구조·몸 안 갈림), §5(금지어), §11(무엇을 좁혀라/절차는 줄이지 마라), §13(스킬 충실)

## 0. v1 → v2 무엇이 바뀌나

v1은 "다음 한 걸음 + 진행 보드"만 있었다. 매일 열어도 **"오늘 내가 뭘 했고, 얼마나 왔고, 어디서 막혔는지"가 한눈에 안 보였다.**
v2는 그걸 채운다: **오늘 칸 · 추천(조언) · 세션별 진도 · 병목 분포**를 더하고, 덜 보는 표·점검은 접는다.

또한 v1 이후 손으로 직접 박았던 "누가 무엇으로 표 + 점검 4칸"은 spec에 없던 상태였다 → 이 문서가 그 둘을 정식으로 포함하고, 접힌 영역으로 정리한다.

## 1. 목적 (왜 만드나)

매일 한 번 열어 **한 화면에서** 다음을 본다:
1. 오늘 내가 **한 일** / **할 일**
2. 내 **진도와 속도** (세션별 + 전체 병목)
3. **추천** — "이건 늦었다 / 이걸 먼저 하라"는 조언

성공 기준: 열면 **스크롤 없이 위쪽만 봐도** "어제까지 뭐 했고, 오늘 뭐 하면 되고, 어디가 막혔는지"가 잡힌다.

## 2. 안 하는 것 (YAGNI)

- 로그인·회원·DB·서버 없음 (열 때 보는 정적 HTML)
- 실시간 갱신 없음 (스크립트 돌릴 때 새로 그림)
- AI를 매번 호출하지 않음 — 추천은 **규칙 기반**(파일 날짜·단계로 스크립트가 판단)
- 매일 손으로 일지 적기 없음 — "오늘 한 일"은 파일 날짜로 자동 수집
- 발행 여부는 발행 목록 파일(`하루끝-키트/발행.md`)로 추적 — 발행 시 한 줄 기록(AI가 사이트 배포할 때 자동 기록 가능). 그 외 단계는 파일 존재로 자동 판정.

## 3. 화면 구성 (위 → 아래)

1. **제목 + 파이프라인** — 상태 → 재료 → 질문지 → 대본 → AI티빼기 → 결점검 → 발행
2. **오늘 칸** — 큰 카드. ✅ 오늘 한 일(오늘 수정된 파일) / ▶ 오늘 할 일(다음 한 걸음)
3. **💡 추천** — 콜아웃. 규칙으로 뽑은 조언 최대 두 줄 (§5 규칙)
4. **세션별 진도** — 각 항목이 7단계 중 어디까지 왔는지 막대 + "N일째" 배지. 늦은 건 ⏰
5. **어디서 막히나(병목)** — 단계별 개수 막대 + 진행률(녹음까지 간 비율)
6. **진행 보드** — 3칸(정리된 재료 / 대본 초안 / 녹음·발행), 각 칸에 항목 목록 (v1 유지)
7. **숫자 요약** — 모은 재료 / 만든 대본 / 녹음까지 (v1 유지)
8. **▸ 세션 생산 라인 — 누가 무엇으로** — 접힘(`<details>`). 펼치면 7단계 표(단계/누가/무엇/스킬·도구/저장위치)
9. **▸ 내보내기 전 점검 4칸** — 접힘(`<details>`). 펼치면 체크박스 4개(브라우저에 저장)

## 4. 핵심 개념 — "항목"과 "단계"

세션을 자동으로 한 줄로 묶기는 어렵다(재료 이름 "김경일_이질문" 과 대본 이름 "텅빈저녁" 은 연결 정보가 없다). 그래서 v2는 **재료·대본 각각을 독립 "항목"으로 보고, 그 항목이 도달한 최고 단계를 진도로 표시**한다.

단계 매핑 (7단계 중 파일/목록으로 잡히는 지점):
- 재료만 있음 (`자료함/정리/*.md`) → **2/7 (재료)**
- 대본 초안 있음 (`outputs` 의 대본 .md, m4a 짝 없음) → **4/7 (대본)**
- 녹음됨 (대본 .md + 짝 m4a 있음) → **6/7 (녹음)**
- 발행됨 (항목 이름이 `발행.md` 목록에 있음) → **7/7 (발행)**
- 발행 판정은 다른 단계보다 우선: `발행.md` 에 이름이 있으면 녹음 여부와 무관하게 7/7.

진행률(병목용) = `발행 항목 수 / (재료+대본+녹음+발행 항목 수)`. "발행까지 간 비율".

## 5. 추천(조언) 규칙 — 규칙 기반, 최대 2줄

순서대로 검사해 해당하는 것 최대 두 줄을 보여준다:
1. **늦은 것 깨우기:** 발행 전 항목(재료·대본·녹음 단계) 중 가장 오래 멈춘 것이 7일↑ 이면 → "「○○」이 N일째 〈단계〉에서 멈췄어요. 〈다음 행동〉 추천." (다음 행동: 재료 단계→"대본 쓰기", 대본 단계→"녹음", 녹음 단계→"발행")
2. **병목 짚기:** 단계별 개수에서 한 단계가 다음 단계보다 2배 이상 많으면 → "재료가 대본보다 빨리 쌓이고 있어요(병목: 〈다음 단계〉)."
3. 둘 다 해당 없으면 → 기존 "다음 한 걸음" 문구 하나만.

추천 문구에도 §5 금지어(치유·회복·성장·변화·코칭·명상·소마틱·이완 등)를 쓰지 않는다.

## 6. 데이터 출처 & 분류 규칙 (v1 유지)

- **재료**: `자료함/정리/*.md`
- **대본·녹음**: `하루끝-키트/outputs/`
  - `.md` 중 `오디오대본*`/`세션*` = 대본, `.m4a` = 녹음본
  - 대본 키와 m4a 키를 정규화(공백·기호·확장자 제거, NFC)해 짝이 맞으면 "녹음", 아니면 "초안"
- macOS 한글 파일명은 `.normalize('NFC')` 로 비교
- `.` / `_` 로 시작하는 파일 제외
- **발행 목록**: `하루끝-키트/발행.md` — 발행한 항목 이름을 한 줄씩(예: `- 오디오대본_하루끝-나에게_v0.4`). 비교는 정규화(NFC + 공백·기호 제거)로 느슨하게 매칭. 파일이 없으면 발행 0개로 취급(에러 아님).
- **오늘 한 일**: 위 폴더들에서 수정시각이 **오늘(한국시간 자정 이후)**인 파일. 한국시간 = `Asia/Seoul` 기준 날짜 비교.

## 7. 동작 방식 (§0 — 내 손 안 감)

- 파일: `automations/session-dashboard.mjs` (Node ESM, 외부 의존성 없음 — 표준 모듈만)
- 실행: `node automations/session-dashboard.mjs`
- 결과: `하루끝-키트/현황판.html` 을 매번 새로 그림
- 점검 체크박스만 브라우저 `localStorage` 에 저장(인라인 `<script>`). 그 외 JS 없음.

## 8. 구조 (작은 순수 함수로 — §13 테스트 가능하게)

부수효과(폴더 읽기·파일 쓰기)와 순수 함수를 분리. 순수 함수는 `node:test` 로 검증.
- 기존 유지: `normalizeName`, `classifyOutputs`, `daysSince`, `findStalled`, `decideNextStep`, `renderHtml`
- v2 신규(순수 함수):
  - `stageOf(item, 발행keys): { label, step }` — 항목의 단계와 7분의 몇인지 (발행=7, 녹음=6, 대본=4, 재료=2). 항목 이름 정규화 키가 `발행keys`에 있으면 발행 우선.
  - `parsePublished(text): Set<string>` — `발행.md` 내용에서 정규화된 발행 키 집합 추출 (`- ` 글머리·공백 무시)
  - `collectToday(items, todayStr): string[]` — 오늘 수정된 항목 이름들
  - `bottleneck({재료, 초안, 녹음}): { counts, rate, 쌓인단계 }` — 병목 분포·진행률
  - `decideAdvice({items, counts}): string[]` — §5 추천 규칙, 최대 2줄
- 신규 렌더 헬퍼: 오늘 칸·진도 막대·병목 막대·접힘 영역

## 9. 톤 & 금지어 (§4·§5)

- 담백·여백. "늦은 것"은 다그치지 않고 조용한 추천으로.
- 금지어 안 씀(추천 문구·점검 설명 포함). 점검 설명에 금지어 단어 자체를 예시로도 쓰지 않는다.

## 10. 검증 (만든 뒤 확인)

- `node --test` 로 순수 함수 전부 통과 (기존 + 신규)
- 실제 실행 시 에러 없이 `현황판.html` 생성
- 한글 파일명 NFC 매칭 정상
- "오늘 한 일"이 실제 오늘 수정 파일과 일치(한국시간 기준)
- 세션별 진도 단계가 파일 상태와 맞음(재료=2/7, 대본=4/7, 녹음=6/7, 발행=7/7)
- `발행.md` 에 적은 이름이 7/7로 잡히고, 파일 없을 때 에러 없이 발행 0개로 동작
- 추천 두 줄이 실제 상태와 맞음 (늦은 것 / 병목)
- HTML에 금지어 없음 (테스트로)
- 접힘 영역(`<details>`)이 평소 접혀 있고 펼쳐짐
