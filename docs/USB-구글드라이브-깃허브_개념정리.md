# USB · 구글드라이브 · GitHub — 천천히 읽는 개념 정리

> 2026-06-30. 윤미선이 "지금은 정확히 이해 못 했지만 나중에 보고 이해하고 싶다"고 해서 만든 글.
> 어려운 말은 한 줄 풀이를 붙였어요. 급하면 맨 아래 **"한 장 요약"** 만 봐도 돼요.

---

## 0. 한 줄로 먼저
- **일반 문서 폴더**(한글·PDF·사진) → **구글드라이브** 에 둬도 돼요. 잘 돼요. 지금처럼 쓰세요.
- **이 '이어' 폴더**(태오 CAB 수업 프로젝트) → **GitHub** 가 집이에요. 태오가 이미 그렇게 연결해놨어요.
- 그래서 **USB는 더 안 들고 다녀도 돼요.** (각 컴퓨터에 한 번씩 내려받기만 하면)

---

## 1. 세 가지가 각각 뭔가

### USB (지금 F: 막대기)
- **물리적인 막대기.** 한 번에 컴퓨터 한 대에만 꽂아 써요.
- 장점: 단순함. 인터넷 없어도 됨.
- 단점: **잃어버리거나 고장 나면 끝.** 백업(사본)이 없어요. 그리고 들고 다녀야 해요.

### 구글드라이브 (G:)
- **인터넷 위의 내 폴더.** 컴퓨터에서 파일을 바꾸면 자동으로 인터넷에 올라가고, 다른 컴퓨터에도 자동으로 내려와요.
- 장점: **자동.** 가만 둬도 동기화됨. 과거 버전도 자동 보관(= 드라이브의 "버전 이력").
- 잘 맞는 것: **그냥 파일 모음** — 한글, PDF, 사진, 워드.

### GitHub (인터넷 저장소)
- **작업 프로젝트 전용 창고.** 글·코드뿐 아니라 "**누가 언제 뭘 바꿨다**"는 작업 기록까지 같이 보관해요.
- 컴퓨터마다 사본을 두고, **"올리기(push)"**, **"받기(pull)"** 로 인터넷 창고와 맞춰요.
- 잘 맞는 것: **이 '이어' 폴더 같은 프로젝트** — 비밀 키 + 작업 기록 + 자동화가 든 것.

---

## 2. 왜 일반 폴더는 드라이브여도 되고, 이 폴더는 GitHub인가

폴더 종류가 다르기 때문이에요. **옷은 옷장에, 음식은 냉장고에** 넣는 것과 같아요. 둘 다 "보관"이지만 넣는 데가 달라요.

이 '이어' 폴더엔 일반 폴더엔 **없는 세 가지**가 들어 있어요:

1. **비밀 키 파일** (`.env`, `creds.yml` 등)
   - GitHub는 "이건 절대 안 올림" 목록(`.gitignore`)을 읽어서 비밀을 자동으로 빼줘요. (이미 잘 설정돼 있어요.)
   - **구글드라이브는 그 목록을 안 읽어요.** 폴더에 있는 건 전부 올려요 → 비밀 키가 구글 서버로 새 나가요.

2. **작업 기록** (숨김 `.git` 폴더)
   - 수백 개 작은 파일이 서로 얽힌 예민한 기록 장치예요.
   - 구글드라이브가 두 컴퓨터로 왔다갔다 동기화하다 어긋나면 **이 기록이 깨질 수 있어요.**
   - (걱정 마세요 — 깨져도 선생님 문서 .md 자체는 멀쩡해요. 다치는 건 '작업 기록' 쪽이에요.)

3. **자동화** (태오가 만든 "GitHub Actions")
   - **GitHub에 올려야만 작동하는** 자동 프로그램이에요. 그래서 이 폴더는 애초에 GitHub용으로 만들어졌어요.

> 정리: 일반 문서엔 이 세 가지가 없으니 드라이브가 딱 맞고, 이 폴더엔 셋 다 있으니 GitHub가 집이에요.

---

## 3. "그럼 USB 계속 꽂았다 뺐다 해야 돼?" → 아니요

지금은 폴더가 **USB 안에만** 있어서 막대기를 들고 다녀야 했어요.
GitHub를 쓰면 이렇게 바뀌어요:

```
            ┌─────────────────────────┐
            │   GitHub (인터넷 창고)   │  ← 진짜 원본이 사는 곳
            │   CAB-STARTER 저장소     │
            └───────────┬─────────────┘
                받기↑ ↓올리기   (인터넷이 옮겨줌)
          ┌─────────────┴─────────────┐
          │                           │
   ┌──────┴──────┐            ┌────────┴──────┐
   │  회사 PC    │            │    맥북       │
   │  (C: 사본)  │            │  (자기 하드)  │
   └─────────────┘            └───────────────┘
```

- 회사 PC 앉으면 → **받기(pull)** 로 최신 내려받고 → 작업하고 → **올리기(push)**.
- 맥북 가면 → **받기(pull)** → 작업 → **올리기(push)**.
- **인터넷이 대신 옮겨주니, 막대기를 들고 다닐 필요가 없어요.**

> **pull(받기)·push(올리기)는 태오가 지난 시간에 가르쳐준 그것** 이에요. 그리고 이 작업은 **AI(클로드)가 대신** 눌러줘요. 선생님은 git이 뭔지 몰라도 돼요.

---

## 4. USB 졸업하려면 — 딱 한 번만

지금은 폴더가 USB에만 있으니, **각 컴퓨터 자기 하드에 사본을 한 번씩 내려받아** 두면 돼요. (이걸 "복제 / clone" 이라고 해요.)

1. 지금 이 PC의 작업을 GitHub에 **올리기(push)** — (아직 안 올린 작업분이 있음)
2. 회사 PC의 C: 같은 데로 한 번 **내려받기(clone)**
3. 맥북에서도 한 번 **내려받기(clone)**

이 한 번만 해두면, 그다음부턴 USB 없이 pull/push로만 이어져요.
**USB는 버리지 말고 여분 백업으로만** 두면 돼요 (들고 다닐 필요 없음).

---

## 5. 모르는 단어 한 줄 풀이
- **저장소(repository, 레포)** = GitHub에 있는 프로젝트 폴더 하나.
- **clone(클론·복제)** = GitHub 창고의 사본을 내 컴퓨터에 처음 한 번 통째로 내려받는 것.
- **pull(풀·받기)** = 창고의 최신 변경을 내 컴퓨터로 가져오는 것.
- **push(푸시·올리기)** = 내 컴퓨터의 변경을 창고로 올리는 것.
- **commit(커밋)** = "여기까지를 한 묶음으로 기록" 하는 것. (작업 기록의 한 칸)
- **.gitignore** = "이 파일들은 GitHub에 올리지 마" 목록. 비밀 파일을 지켜줌.
- **GitHub Actions** = GitHub에 올려두면 자동으로 도는 프로그램(예: 매일 대시보드 갱신).

## 한 장 요약
| | USB(F:) | 구글드라이브(G:) | GitHub |
|---|---|---|---|
| 자동 동기화 | ✕ (들고 다녀야) | ◎ 자동 | △ pull/push (AI가 대신) |
| 잃어버려도 복구 | ✕ 백업 없음 | ◎ 클라우드 사본 | ◎ 인터넷 창고 |
| 비밀 키 안전 | — | ✕ 새 나감 | ◎ 자동 제외 |
| 작업 기록 보존 | △ | ✕ 깨질 수 있음 | ◎ 안전 |
| 잘 맞는 것 | 잠깐 옮기기·백업 | 일반 문서 | **이 프로젝트 폴더** |

**결론:** 이 '이어' 폴더는 이미 GitHub에 연결돼 있어요(태오가 세팅). 구글드라이브로 옮기지 말고, 각 컴퓨터에 한 번씩 내려받아 두면 USB 없이 pull/push로 이어져요.
