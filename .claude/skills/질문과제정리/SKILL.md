# 질문·과제 정리 (AI 대화·자료 → 출시 재료)

윤미선의 매일 반복업무 1순위를 자동화한다: **AI와 나눈 대화나 읽은 자료를, 출시로 이어지는 재료로 바꾼다.**
통증은 "고민·탐색에 시간이 쏠려 출시가 늦는 구조"다. 이 스킬의 목적은 흩어진 대화·자료를 **손에 잡히는 4덩어리**로 정돈해, 미선이 *판단과 최종 문장*에만 시간을 쓰게 하는 것.

## 무엇을 받나 (입력)
다음 중 무엇이든 받는다. 사용자가 형식을 안 맞춰도 알아서 처리한다.
- ChatGPT·Claude 대화 붙여넣기 (길어도 됨)
- 읽은 자료: 기사·블로그·책 발췌·인터뷰·**영상 자막(트랜스크립트)**·메모
- 폴더 안 파일 경로 (예: `자료함/...`, `하루끝-키트/inputs/...`)

입력이 비었거나 모호하면 **딱 한 가지만** 묻는다: "무엇을 정리할까요? (붙여넣거나, 폴더 안 파일 경로를 알려주세요)" — 길게 캐묻지 않는다(시간이 묶인 사용자다).

## 무엇을 내나 (출력 — 항상 이 4덩어리)
아래 순서·제목을 그대로 쓴다. 각 덩어리가 왜 있는지 알고 채운다.

### 1. 요약
3~6줄. 원자료가 무슨 말을 하는지 담백하게. 군더더기·미사여구 없이.

### 2. 핵심 아이디어
불릿 3~7개. 자료에서 **건질 만한 생각**만. 각 줄은 한 문장. 미선의 사업(혼자 쓰는 감각·기록·질문 도구)과 닿는 지점이 보이면 끝에 `→ 도구 연결: …` 한 조각을 붙인다.

### 3. 질문 후보 (고객 도구용 · 가장 중요)
미선의 도구에 **실제로 넣을 수 있는 질문** 3~7개. 이건 **고객이 읽는 글**이므로 규칙이 가장 엄격하다(아래 〈고객용 규칙〉). 자기 자신에게 던지는, 답이 정해지지 않은, 머무르게 하는 질문으로.
- 좋은 예: "오늘 하루 중, 가장 나 같았던 순간은 언제였나요?"
- 좋은 예: "지금 몸에서 가장 무거운 곳은 어디인가요?"
- 피할 것: 지시·평가·정답이 있는 질문 ("오늘 목표를 달성했나요?")

### 4. 실행과제 (내 출시 작업)
미선이 **출시를 향해 실제로 할 일** 3~6개. 이건 *내부용*이라 톤·금지어 규칙에서 자유롭다(일반 업무 언어 OK).
- 각 과제는 동사로 시작하고 손에 잡히게. (예: "〈하루 끝〉 워크북에 '무거운 곳' 질문 1개 추가")
- §0 위반(몸이 갈리는 구조·없는 고객용 인프라·SNS 운영 확대)인 과제는 **넣지 말고**, 대신 "안 하는 게 맞아 보임: …" 한 줄로 짧게 알린다.

## §0 — 모든 정리 위 원칙 (먼저 통과)
정리하기 전에 두 질문을 통과시킨다. 자료가 여기 어긋나면, 억지로 도구에 끼우지 말고 **"이 자료는 우리 결과 먼〔이유〕"** 를 1~2줄로 정직하게 적고 넘어간다.
1. **이것이 사람들이 자기 자신과의 대화를 이어가게 만드는가?**
2. **윤미선의 몸이 갈리지 않는 구조인가?** (미선의 시간이 계속 투입돼야만 유지되면 과제로 내지 않는다)

## 고객용 규칙 (3번 질문 후보에만 엄격 적용)
질문 후보는 고객이 읽으므로, 미선의 결을 반드시 지킨다. 전체 기준은 `하루끝-키트/seeds/yunmiseon_seed.md` 와 루트 `CLAUDE.md`. 핵심만 옮기면:

**톤:** 담백 · 정갈 · 조용한 깊이. 시적이되 과장 없음. 결론을 서둘러 내리지 않고 여백을 남긴다. 개념어보다 *사용 장면*으로 말한다.

**좋아하는 말(이 결로):** 이어감 · 기록 · 감각 · 질문 · 문장 · 머무름 · 정돈 · 숨 · 무게 · 온도 · 느낌 · 리듬 · 흔적 · 여백 · 방향 · 작은

**절대 금지어 (YOU MUST — 고객용 산출물에서 빼고, 좋아하는 말로 바꾼다):**
- 개념어: 치유 · 회복 · 성장 · 변화 · 자기계발 · 자기혁신 · 코칭 · 클래스 · 교육 · 스튜디오 · 리트릿 · 챌린지 · 목표달성 · 생산성
- 장르 이름(본질이 그렇더라도 단어를 입에 올리지 않는다): 명상 · 소마틱 · 바디스캔 · 이완 · 마음챙김 · 호흡법 → 대신 일어나는 일 그대로: 숨 · 몸 · 무게 · 온도 · 머무름 · 느낌
- 과장·압박: "당신도 할 수 있습니다" · "인생이 바뀝니다" · "성공의 비밀" · "○○일 만에" · "기적" · 동기부여 과잉

**자가 점검(출력 직전 필수):** 3번 질문 후보를 다시 읽고, 위 금지어가 있으면 좋아하는 말로 바꾼다. 출력 맨 끝에 한 줄로 `금지어 점검: 통과` 또는 `금지어 점검: '성장'→'이어감'으로 바꿈` 처럼 무엇을 바꿨는지 적는다(미선이 신뢰하고 넘어갈 수 있게).

## 어디에 저장하나
정리 결과를 **채팅에 보여주고**, 동시에 **파일로 저장**한다.
- 저장 위치: `자료함/정리/YYYY-MM-DD_<주제짧게>.md` (폴더 없으면 만든다)
- 파일 맨 위에 원자료 출처(붙여넣기면 "붙여넣기 대화", 파일이면 경로, 영상이면 링크)와 날짜를 적는다.
- 노션 저장은 **사용자가 "노션에도"라고 할 때만** 한다(매번 자동으로 바깥에 올리지 않는다). 올릴 땐 `notion` MCP 사용, "참고 유튜브 영상" 같은 기존 페이지나 사용자가 지정한 곳 아래에.

## 일하는 방식 (짧고 집중적으로)
- 미선은 평일 본업으로 시간이 묶여 있다. **되묻기를 최소화**하고, 합리적 기본값으로 바로 정리한다.
- 최종으로 고객에게 닿는 **단 하나의 문장**은 미선이 직접 고른다. 그러니 질문 후보는 *후보로* 충분히 주되, "이 중 골라 다듬으세요"의 자세로.
- 확신이 없으면 지어내지 말고 "확인 안 됨"이라고 적는다.
