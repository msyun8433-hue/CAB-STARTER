# 1주차 보고서 점검 메모
작성: 2026-06-21 · 대상: `week1_report.md` · 기준: 1주차 강의 요구사항

> 보고서 본문은 건드리지 않았다. 여기 **빠진 것·보강 제안**만 모았다. 붙일지 말지는 윤미선이 정한다.

---

## A. 강의가 1주차에 요구한 것 vs 내 보고서
강의 "손에 잡힐 3가지" + 오늘 4단계 기준.

| 강의 요구 | 했나 | 보고서에 있나 | 비고 |
|---|---|---|---|
| ① CLAUDE.md 1개 | ✅ 함 | △ 암시만 | 보고서가 CLAUDE.md를 직접 언급 안 함. 한 줄 있으면 좋음 |
| ② X-mind 8영역 지도 1장 | ✅ 함 (`business/02-8부서.md` = 8부서+반복업무+우선순위 매트릭스) | ❌ 언급 없음 | 사실은 했는데 보고서엔 안 보임. 발표 때 손해 |
| ③ 4주간 자동화 영역 1개 | ✅ 함 | ✅ 1번 섹션 | 단, **아래 C 불일치 주의** |
| MCP 연결 | △ notion만 | ❌ 언급 없음 | `.mcp.json`엔 notion 1개. 강의는 5개 권유 |
| 폴더 이름 변경 | ❌ 아직 | ✅ 6번에 솔직히 | OK |
| GitHub 로그인 | ❌ 아직 | ✅ 6번에 솔직히 | OK |

**결론:** 보고서는 솔직하고 좋다. 다만 **실제로 한 것(8영역 지도·MCP)이 보고서에 안 적혀서, 한 걸 안 한 것처럼 보임.** 이게 제일 큰 보강 포인트.

---

## B. 붙이면 좋은 보강 문구 (그대로 복붙 가능)

**(1) 4번 "실제로 만든 것" 끝에 한 줄 — 빠진 산출물 살리기**
```
- 이 밖에: 내 사업을 8영역으로 펼친 지도(business/02-8부서.md) + CLAUDE.md(내 사업 매뉴얼) + 노션 MCP 연결까지 했다.
```

**(2) 6번 "안 한 것"에 MCP 한 줄 추가 — 솔직하게**
```
- MCP는 노션만 연결. 강의에서 권한 5개(supabase·github·google·playwright)는 필요해질 때 붙일 예정.
```

> 두 줄만 넣으면 "손에 잡힐 3가지(CLAUDE.md·X-mind 지도·자동화 영역)"가 보고서에서 다 보인다.

---

## C. ⚠️ 점검 중 발견한 진짜 쟁점 — "자동화 영역" 두 개가 다름
강의 기준으로 보면 내 보고서와 `business/02-8부서.md`가 **서로 다른 걸 1순위라고 말하고 있다.**

| 문서 | 1순위로 적힌 것 | 정체 |
|---|---|---|
| `week1_report.md` (1번) | "셀프 세션 만드는 흐름" | **제품 제작** |
| `business/02-8부서.md` | "AI 대화·자료 → 질문·과제 자동 정리" | **반복업무 자동화** |

- 강의가 말한 '자동화 영역'은 **효과×노력 매트릭스로 고른, 내 반복 노동을 줄이는 것** = 후자(반복업무 자동화)에 가깝다.
- 내가 1주차에 실제로 한 건 **제품(셀프세션 키트)을 끝까지 만들어 출시**한 것 = 전자.
- 즉 **나는 '제품'을 했고, 강의 틀의 '자동화 영역'은 '반복업무 자동화'를 가리킨다.** 둘 다 가치 있지만 **같은 게 아니다.**

→ 이건 틀렸다기보다 **태오에게 물어볼 핵심 질문**이다. (`week1_태오질문_다듬음.md` 1번으로 옮겨 날카롭게 정리함)
→ 결정 나기 전까지 두 파일을 억지로 맞추지 말 것. 어느 쪽이 4주 중점인지는 태오 답 듣고 정한다.
