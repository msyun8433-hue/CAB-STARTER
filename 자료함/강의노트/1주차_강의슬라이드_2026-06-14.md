## 0. 마음가짐

## 1. 4주의 약속
- ❌ 4주 안에 사업 전체 자동화 (무리하면 끝까지 못 감)
- ✅ **자동화 한 가지를 끝까지 완성 + 스스로 만드는 역량** → 졸업 후 혼자 두 번째·세 번째
- 완성된 도구를 주는 게 아니라 **스스로 만드는 방법**을 익히는 4주.

## 2. 자동화 5단계 레벨
1. **열정가** — 검색창처럼 씀 (하루 30분 절약)
2. **초보자** — 간단한 일을 시켜봄 (주 5h+) ← 1주차 도착점
3. **중급자** — 내 폴더를 직접 맡김 (주 10h+) ← 2~3주차
4. **고급자** — 반복 일을 자동화 (병렬 팀) ← 4주차 졸업
5. **아키텍트** — 대부분 자동·나는 검수만 ← 졸업 후
- CAB 4주 = 레벨 1~2 → 레벨 4. 레벨 5는 졸업 후 스스로.

## 6. Claude Code 개념
- 챗봇(ChatGPT·Gemini)은 **대화창 안에서 답만** 함. 내 파일·일정은 못 봄.
- **Claude Code = 내 컴퓨터에서 직접 일하는 신입 직원** — 파일을 읽고·고치고·저장하고, 폴더를 만들고 정리. 자리를 비워도 굴러감.
- 예: "이 폴더 메모들 주제별로 정리해줘" → ChatGPT ❌ / Claude Code ✅
- **4가지 모드 (shift+tab 전환):** 🤚물어보고(기본·안전) · ⚡자동편집 · 🚀오토 · 📋플랜모드. 처음엔 '물어보고'.

### 순서가 핵심
**① 매뉴얼(CLAUDE.md) → ② 도구(MCP·Skill·Plugin·API) → ③ 에이전트(자동화)**
- 신입에게 매뉴얼·도구 없이 일 맡기면 꼬인다. **에이전트보다 사업 구조가 먼저.**
- 무작정 한 번에 다 자동화하면 서로 엉킴 → **오늘은 딱 1개부터.**

### Claude Code 5요소
📋 CLAUDE.md(매뉴얼) · ⚡ Workflow · 🤖 Agent(전문 인턴) · 🔌 Tool(내장+MCP) · ⚙️ settings.json(권한·훅)

## 7. CLAUDE.md
- AI에게 '내가 누군지' 알려주는 **한 장짜리 문서**. 명령할 때마다 **매번 먼저 읽음.**
- `.md` = 마크다운(글로 적는 텍스트 문서. `#` 제목, `-` 목록).
- 루트에 메인 1개, 하위 폴더마다 추가 가능(작업 시 덧붙어 우선).
- 우리는 개발이 아니라 **'내 사업'을 Claude 머릿속에 넣는 것.**
- **잘 구성할수록** 내 사업·고객·톤을 정확히 반영.

### 담기는 9가지 (= 인터뷰 9질문 = /init 섹션)
1. 사업 한 줄 + 사명 · 2. 꿈의 고객 한 명 · 3. 상품·가격·일정 · 4. 말투·톤 · 5. 절대 안 쓸 단어 · 6. §0 의사결정 원칙 · 7. 우선순위 3개 · 8. 이번 분기 목표 · 9. 안 하는 것

### 작성 원칙 (Anthropic 공식)
- **80~100줄 목표, 200줄 최대.** "이 줄을 빼면 Claude가 실수하나? 아니면 지워라."
- 비대해지면 정작 중요한 지시를 무시함.

## 8. 폴더 구조 (cab-starter)
```
내 사업 폴더/
├─ CLAUDE.md        인터뷰+/init 로 채워짐 (처음엔 없음=정상)
├─ business/        사업 자료·인터뷰 답변
├─ workbook/W1.md   이번 주 워크북
├─ automations/     내가 만든 자동화
└─ .claude/         Claude 설정·도구 칸 (점=숨김)
    ├─ settings.json               권한·안전 기본값
    ├─ commands/cab-interview.md   /cab-interview 명령
    ├─ agents/  skills/  rules/
```
- 윗단 = 사람이 보는 **사업 자료** / `.claude/` = **Claude 부리는 법**.

### .claude/ 4가지 도구 (개념만, 만들기는 2주차)
- **commands** — 내가 `/이름` 으로 부르는 단축 명령
- **agents** — 따로 떨어져 알아서 일하는 전문 인턴 (`@이름` / 자동 위임)
- **skills** — 참고 자료까지 묶은 작업 능력 (`/이름` / 자동)
- **rules** — 그 파일 만질 때만 조용히 뜨는 상황별 규칙
- **settings.json** — 허용(allow)·물어보기(ask)·차단(deny). cab-starter에 안전값 포함.

## 11. MCP & Context

### 외부 도구 붙이는 두 방법
- 🔌 **MCP** = "표준 USB". `claude mcp add OO` 또는 "OO 연결해줘" 한 마디. 코딩 X. 읽기·간단작업(노션·드라이브·메일). **→ 오늘 이걸로.**
- 🔧 **API** = "도구마다 직접 배선". 코딩 O. 업로드·자동발행(영상·인스타). **→ 2주차.**
- MCP = AI와 노션·구글·슬랙 사이의 **콘센트 어댑터.**

### 오늘 깔 MCP 5개
`notion`(문서·DB) · `supabase`(DB·백엔드) · `playwright`(브라우저 조작) · `github`(코드 저장소) · `google`(드라이브·메일·캘린더)
- 연결 후 `/mcp` 로 인증 상태 확인 → 브라우저 로그인·권한 허용 → 테스트 한 마디.
- "꼭 깔 필수 세트"는 없음. **내 일에 필요한 것만.**

### Context(컨텍스트) = Claude의 '작업 기억'
- 토큰으로 참(영어 1토큰≈4자, 한국어 2~4배). 기본 약 **200,000 토큰**, 강한 모델 1,000,000.
- 제일 많이 먹는 것 = **읽은 파일** (100KB ≈ 25,000 토큰). → 콕 집어 읽히기.
- 차면(80%+) 기억 흐려지고 실수↑ → **정리 + 보내기 전 사람 확인.**
- **관리 도구 3개:** `/clear`(주제 바뀔 때 비우기) · `/compact`(길어졌을 때 요약, 60~70%서 직접·80~90%서 자동) · `/context`(지금 얼마나 찼나 확인)
