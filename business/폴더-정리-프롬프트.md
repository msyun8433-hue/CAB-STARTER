# 내 사업 폴더 정리 프롬프트 (3층 구조 만들기)

> 자동화 파일이 한곳에 섞여 어지러울 때, 아래 블록을 **순서대로** Claude Code 에 붙여넣으세요.
> **핵심 원칙: 바로 옮기라고 하지 말 것.** 진단 → 제안 → 내 승인 → 이동 순서로 가야 안 꼬여요.

## 목표 구조 — 3층으로 쌓기

```
내-사업/                       ← 최상위 (git repo 하나)
├─ CLAUDE.md                   ← 사업 공통: 톤·고객·금지 (9가지)
├─ .claude/
│   ├─ settings.json           ← 공통 권한
│   └─ agents/ · skills/        ← 사업 전체가 공유하는 인턴·도구
├─ 콘텐츠-5채널/                ← 자동화 1
│   └─ CLAUDE.md               ← 이 시스템만 (7가지 구성)
├─ 고객-응대/                   ← 자동화 2
│   └─ CLAUDE.md
└─ 예산서-작성/                 ← 자동화 3
    └─ CLAUDE.md
```

- 자동화 폴더에서 Claude Code 를 켜면 **3층(전역 + 사업 + 그 자동화)을 다 합쳐 읽어요.**
- 그래서 사업 톤·고객은 **최상위에 한 번만**, 자동화 세부는 각 폴더에.
- 우선순위 = **가까운(구체적) 게 이김** (자동화 하위 > 사업 루트 > 전역 `~/.claude/`).

---

## 0단계 · 안전장치 (먼저 한 줄)

```
지금 폴더 상태 그대로 git에 커밋해줘. 정리하다 꼬이면 되돌리게.
```

## 1단계 · 진단 + 구조 제안 (아직 안 옮김)

> 큰 작업이니 **Plan Mode (Ctrl+G)** 로 켜고 시키면 더 안전해요.

```
이 폴더가 자동화 스크립트랑 파일이 한곳에 섞여서 어지러워.
바로 옮기지 말고, 먼저 이것부터 해줘:
1. 폴더 전체를 훑어서 어떤 자동화·작업들이 있는지 목록으로 정리
2. 비슷한 것끼리 묶어서 하위 폴더로 나누는 구조를 제안
   (예: 콘텐츠-자동화/ · 고객-응대/ · 문서-작성/)
3. 어떤 파일을 어느 폴더로 옮길지 표로 보여줘
아직 옮기지 말고 제안만. 내가 보고 OK 하면 그때 옮기자.
```

## 2단계 · 승인 후 재배치 + 각 CLAUDE.md 생성

```
좋아 그 구조대로 가자.
1. 제안한 하위 폴더들 만들고 파일을 거기로 옮겨줘
2. 각 하위 폴더에 그 자동화 전용 CLAUDE.md 를 만들어줘
   (개요=이 폴더가 자동화하는 반복 1개 · 기술스택 · 금지사항 · 워크플로우)
3. 최상위 CLAUDE.md 에는 사업 공통(톤·고객·금지)만 남기고
   세부는 하위로 내려줘
옮기기 전에 뭐가 어디로 가는지 한 번 더 요약해주고 시작해.
```

## 3단계 · 앞으로 새 자동화 추가할 때 (한 줄 루틴)

```
새 자동화 [이름] 만들 거야. 최상위 아래에 [이름]/ 폴더 만들고,
그 안에 CLAUDE.md(7가지 구성) 먼저 잡아줘.
사업 공통은 최상위에서 상속하니까 여기엔 이 자동화 전용만.
```

---

**왜 단계를 나누나** — "한 번에 다 정리해줘"라고 하면 Claude 가 추측으로 막 옮겨서 더 어지러워져요. 1단계에서 **분류를 내가 먼저 확인**하고 2단계에서 실행하는 게 안전해요. 한 번에 다 말고 **제일 어지러운 한 덩어리부터** 시켜보세요.
